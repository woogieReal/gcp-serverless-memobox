# 배포 및 검증 작업계획

## 목표

CI/CD 파이프라인을 통해 Cloud Functions 배포가 정상적으로 완료되는지 확인하고,
실제 오피스 내부망 환경에서 전체 CRUD 엔드포인트가 기획서 명세대로 동작하는지 검증한다.
IP 차단 동작도 함께 확인하여 보안 통제가 정상 작동함을 보장한다.

## 작업 범위

- GitHub Actions 파이프라인 성공 확인
- 배포된 Cloud Functions HTTPS 엔드포인트 주소 확보
- 오피스 내부망에서 CRUD 통합 테스트
- 외부 IP 차단 확인

---

## 세부 작업 목록

### 1단계: GCP API 활성화 (사전 조건)

- [x] Cloud Functions API 활성화
  - `https://console.developers.google.com/apis/api/cloudfunctions.googleapis.com/overview?project=woogie-sandbox-gcp`

### 2단계: 파이프라인 성공 확인

- [ ] GitHub Actions 재실행 후 초록불(Success) 확인
- [ ] 배포 완료 후 HTTPS 엔드포인트 주소 확보

### 3단계: CRUD 통합 테스트 (오피스 내부망)

- [ ] `GET /memoApi/` — 메모 목록 조회
- [ ] `POST /memoApi/` — 신규 메모 생성
- [ ] `GET /memoApi/:filename` — 메모 내용 조회
- [ ] `PUT /memoApi/:filename` — 메모 수정
- [ ] `DELETE /memoApi/:filename` — 메모 삭제

### 4단계: 예외 케이스 검증

- [ ] 동일 `filename`으로 `POST` 재요청 시 `409` 반환 확인
- [ ] 존재하지 않는 `filename`으로 `GET`, `PUT`, `DELETE` 요청 시 `404` 반환 확인
- [ ] 잘못된 `filename` 패턴(`../`, 특수문자 등) 요청 시 `400` 반환 확인
- [ ] 외부 IP에서 요청 시 `403` 반환 확인

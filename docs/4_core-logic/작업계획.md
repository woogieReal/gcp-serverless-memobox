# 핵심 로직 구현 작업계획

## 목표

메모 CRUD API의 실제 동작 로직을 구현한다.
IP 화이트리스트와 파일명 검증 미들웨어로 보안을 확보하고,
GCS를 데이터 저장소로 활용하는 CRUD 함수를 작성한다.
모든 에러 응답은 `{ "error": "..." }` JSON 형식으로 통일하여 일관성을 보장한다.

## 작업 범위

- IP 화이트리스트 미들웨어 (`middleware/ipWhitelist.js`)
- 파일명 검증 미들웨어 (`middleware/validateFilename.js`)
- GCS CRUD 로직 (`services/gcs.js`)
- Express 라우터 (`routes/memo.js`)

---

## 세부 작업 목록

### 1단계: IP 화이트리스트 미들웨어

- [x] `middleware/ipWhitelist.js` 생성
  - `x-forwarded-for` 헤더에서 마지막 IP 추출
  - 환경변수 `ALLOWED_IPS` (쉼표 구분)에서 허용 IP 목록 로드
  - 허용 IP 목록과 비교 → 불일치 시 `403` 반환

### 2단계: 파일명 검증 미들웨어

- [x] `middleware/validateFilename.js` 생성
  - 허용 패턴 (`^[a-zA-Z0-9_-]+$`) 검증 → 위반 시 `400` 반환
  - 100자 길이 초과 검증 → 위반 시 `400` 반환

### 3단계: GCS CRUD 서비스 구현

- [x] `services/gcs.js` 생성
  - `listMemos` — 버킷 내 파일명 리스트 반환
  - `getMemo` — 오브젝트 텍스트 내용 반환
  - `createMemo` — 중복 확인 후 업로드 (중복 시 `409`)
  - `updateMemo` — 존재 확인 후 덮어쓰기 (미존재 시 `404`)
  - `deleteMemo` — 오브젝트 제거 (미존재 시 `404`)

### 4단계: Express 라우터 연결

- [x] `routes/memo.js` 생성
  - `GET /memoApi/` → `listMemos`
  - `GET /memoApi/:filename` → `getMemo`
  - `POST /memoApi/` → `createMemo`
  - `PUT /memoApi/:filename` → `updateMemo`
  - `DELETE /memoApi/:filename` → `deleteMemo`
  - 에러 응답 형식 `{ "error": "..." }` JSON으로 통일

# GitHub Repository 연동 작업계획

## 목표

GitHub Actions가 GCP에 인증할 수 있도록 서비스 계정 키를 GitHub Environment Secrets에 안전하게 등록한다.
`Production` Environment를 별도로 생성하여 키 노출 범위를 해당 환경으로 제한하고,
이후 CI/CD 워크플로우가 이 Secret을 참조해 자동 배포를 수행할 수 있는 기반을 마련한다.

## 작업 범위

- GitHub Repository에 `Production` Environment 생성
- `terraform/sa-key.json` 내용을 `GCP_SA_KEY` Secret으로 등록

---

## 세부 작업 목록

### 1단계: Environment 생성

- [X] GitHub Repository → Settings → Environments → `New environment`
- [X] 이름: `Production` 으로 생성

### 2단계: Secret 등록

- [X] 생성된 `Production` Environment → `Add secret` 클릭
- [X] Name: `GCP_SA_KEY`
- [X] Value: `terraform/sa-key.json` 파일 내용 전체 붙여넣기

---

## 변경 대상 파일

| 파일 | 변경 내용 |
|------|-----------|
| GitHub Environment Secrets | `GCP_SA_KEY` 등록 (웹 콘솔 작업, 파일 변경 없음) |

---

## 완료 기준

- GitHub Repository Settings에서 `Production` Environment가 확인된다
- `Production` Environment Secrets에 `GCP_SA_KEY` 항목이 등록되어 있다

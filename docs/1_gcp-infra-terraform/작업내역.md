## [2026-06-19] GCP 인프라 세팅 (Terraform)

### 작업 내용
- Terraform v1.15.6 설치 확인 (Homebrew `hashicorp/tap` 경로로 설치)
- `terraform/main.tf` 작성 — GCS 버킷, 서비스 계정, IAM 바인딩, JSON 키 출력 리소스 정의
- `.gitignore`에 Terraform 관련 민감 파일 추가 (`sa-key.json`, `.terraform/`, `*.tfstate`)
- `terraform init` → `terraform plan` → `terraform apply` 순서로 프로비저닝 실행
- GCP에 리소스 6개 생성 완료 및 `terraform/sa-key.json` 로컬 출력 확인

### 주요 결정 사항
- **Terraform 채택:** GCP 콘솔 직접 조작 대신 IaC로 리소스를 코드화하여 재현성 확보. 프로젝트 규모가 작아 Workload Identity Federation 대신 JSON 키 방식 유지하되, 권한을 최소화하는 방향으로 결정.
- **IAM 권한 최소화:** 서비스 계정에 `Cloud Functions Developer`(프로젝트 레벨)와 `Storage Object Admin`(버킷 한정) 두 가지만 부여. 불필요한 광범위 권한 부여 배제.
- **`local` 프로바이더 사용:** `google_service_account_key` 리소스의 private_key를 `base64decode` 후 `local_file`로 출력하여 별도 스크립트 없이 키 파일 자동 생성.

### 이슈 & 해결
- **Homebrew `terraform` formula 없음:** `brew install terraform` 실패. HashiCorp 공식 tap으로 이전되어 `brew tap hashicorp/tap` 후 `brew install hashicorp/tap/terraform`으로 해결.
- **GCP 인증 오류:** `terraform plan` 실행 시 Application Default Credentials 미설정으로 실패. `gcloud auth application-default login`으로 해결.

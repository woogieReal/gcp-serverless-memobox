# 프로젝트 초기 세팅 작업계획

## 목표

Cloud Functions에 배포할 Node.js 프로젝트의 기반을 구성한다.
`package.json` 초기화와 필수 의존성 설치로 실행 환경을 갖추고,
엔트리 파일(`index.js`)을 생성하여 이후 핵심 로직 구현의 출발점을 마련한다.

## 작업 범위

- `package.json` 초기화 및 의존성 설치 (`express`, `@google-cloud/storage`)
- 프로젝트 디렉토리 구조 확정
- 엔트리 파일 `index.js` 생성

---

## 세부 작업 목록

### 1단계: package.json 초기화

- [x] `npm init -y` 로 `package.json` 생성
- [x] `main` 필드를 `index.js`로 설정 확인

### 2단계: 의존성 설치

- [x] `npm install express` — HTTP 라우터
- [x] `npm install @google-cloud/storage` — GCS 연동

### 3단계: 디렉토리 구조 및 엔트리 파일 생성

- [x] 프로젝트 루트에 `index.js` 생성 (Express 앱 초기화 및 Cloud Functions 엔트리포인트)

---

## 변경 대상 파일

| 파일 | 변경 내용 |
|------|-----------|

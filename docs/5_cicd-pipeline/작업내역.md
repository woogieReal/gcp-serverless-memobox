## [2026-06-19] CI/CD 파이프라인 구성

### 작업 내용
- `.github/workflows/deploy.yml` 작성 — `main` 브랜치 push 트리거, GCP 인증 및 Cloud Functions Gen 2 자동 배포 구성
- GitHub `Production` Environment Secrets에 `ALLOWED_IPS` 등록 (`125.129.72.46`)

### 주요 결정 사항
- **`environment: Production` 명시:** 워크플로우에 Environment를 지정하여 `GCP_SA_KEY`, `ALLOWED_IPS` Secret 접근을 `Production` 환경으로 한정. Repository Secrets 대신 Environment Secrets를 쓰는 보안 정책과 일관성 유지.
- **`ALLOWED_IPS`를 Secret으로 관리:** IP 목록을 코드에 하드코딩하지 않고 GitHub Secrets로 주입하여 코드 변경 없이 IP 추가/제거 가능.
- **`BUCKET_NAME`도 `--set-env-vars`로 주입:** 버킷명을 배포 명령어에서 환경변수로 전달하여 `services/gcs.js`의 기본값에 의존하지 않도록 명시적으로 설정.
- **`--allow-unauthenticated` 사용:** GCP 레벨 인증은 비활성화하고 코드 레벨 IP 화이트리스트로 접근을 통제하는 기존 설계 방식 유지.
- **`google-github-actions/auth@v2` 사용:** gcloud SDK 직접 설치 방식 대신 공식 GitHub Action으로 인증 처리하여 설정을 간소화.

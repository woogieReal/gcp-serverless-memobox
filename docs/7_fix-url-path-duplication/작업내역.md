## [2026-07-08] URL 경로 중복 제거

### 작업 내용
- `index.js`의 Express 라우터 마운트 경로를 `/memoApi` → `/`로 변경
- API URL이 `https://...cloudfunctions.net/memoApi/memoApi/` → `https://...cloudfunctions.net/memoApi/`로 단순화됨
- `scripts/test-api.sh` 작성 — 정상 시나리오 7개 + 예외 시나리오 5개, 총 12개 케이스 자동 검증
- 배포 후 테스트 스크립트 실행, 12/12 전 케이스 PASS 확인

### 주요 결정 사항
- **Express 마운트 경로를 `/`로 변경한 이유**: Cloud Functions Gen 2는 요청을 Express에 전달할 때 함수 이름 prefix(`/memoApi`)를 자동으로 제거한다. 따라서 `/memoApi`로 마운트하면 클라이언트가 URL에 `memoApi`를 두 번 써야 하는 구조가 생긴다. 라우터를 `/`에 마운트하면 Cloud Functions URL 구조와 자연스럽게 맞아떨어진다.
- `routes/memo.js` 내부 라우트 정의(`GET /`, `GET /:filename` 등)는 변경 불필요

### 이슈 & 해결
- **`head: illegal line count -- -1` 경고**: macOS의 `head` 명령어가 음수 line count를 지원하지 않아 발생. 테스트 결과에는 영향 없음. 추후 `sed` 등으로 교체 가능.

### 변경 파일
- `index.js` (수정 — 라우터 마운트 경로 `/memoApi` → `/`)
- `scripts/test-api.sh` (신규 — CRUD 및 예외 케이스 자동 검증 스크립트)
- `docs/7_fix-url-path-duplication/작업계획.md` (신규)

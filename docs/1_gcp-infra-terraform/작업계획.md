# GCP 인프라 세팅 (Terraform) 작업계획

## 목표

GCP 콘솔을 직접 조작하는 대신 Terraform으로 인프라를 코드화하여 프로젝트 내에서 리소스를 관리한다.
버킷, 서비스 계정, IAM 권한을 `terraform/main.tf` 하나로 선언하여 `terraform apply` 한 번으로 전체 인프라를 재현 가능하게 만든다.
이후 환경 재구성이나 리소스 변경 시 콘솔 없이 코드 수정만으로 대응할 수 있다.

## 작업 범위

- Terraform 설치 여부 확인
- `terraform/main.tf` 작성 (버킷, 서비스 계정, IAM 바인딩, JSON 키 출력)
- Terraform 초기화 및 인프라 프로비저닝 실행
- 발급된 서비스 계정 JSON 키 파일 확인

---

## 세부 작업 목록

### 1단계: 환경 확인

- [x] `terraform -v` 로 Terraform 설치 확인

### 2단계: Terraform 코드 작성

- [x] `terraform/` 디렉토리 생성
- [x] `terraform/main.tf` 작성
  - GCP 프로바이더 설정
  - GCS 버킷 리소스 정의 (메모 파일 저장용)
  - 서비스 계정(Service Account) 리소스 정의
  - IAM 바인딩 정의
    - `Cloud Functions Developer` 권한 부여
    - `Storage Object Admin` 권한 부여 (생성한 버킷 한정)
  - 서비스 계정 JSON 키 리소스 정의 및 로컬 파일 출력 (`terraform/sa-key.json`)

### 3단계: 프로비저닝 실행

- [x] `terraform init` — 프로바이더 플러그인 초기화
- [x] `terraform plan` — 생성될 리소스 사전 확인
- [x] `terraform apply` — 인프라 프로비저닝 실행
- [x] 출력된 `sa-key.json` 파일 내용 확인

---

## 변경 대상 파일

| 파일 | 변경 내용 |
|------|-----------|
| `terraform/main.tf` | GCP 리소스 전체 정의 (신규 생성) |
| `terraform/sa-key.json` | `terraform apply` 후 자동 생성되는 서비스 계정 키 파일 |
| `.gitignore` | `sa-key.json` 커밋 방지 항목 추가 |

---

## 완료 기준

- `terraform apply` 가 오류 없이 완료된다
- GCP 콘솔에서 버킷과 서비스 계정이 정상 생성된 것을 확인할 수 있다
- `terraform/sa-key.json` 파일이 로컬에 생성된다

# CI/CD 파이프라인 구성 작업계획

## 목표

`main` 브랜치에 push가 발생하면 자동으로 Cloud Functions에 배포되는 파이프라인을 구성한다.
GitHub Actions가 `Production` Environment Secrets에서 GCP 인증 키를 꺼내 `gcloud functions deploy --gen2`로 배포하며,
배포 시 IP 화이트리스트 환경변수(`ALLOWED_IPS`)를 함께 주입한다.
이 파일이 완성되면 이후 코드 변경은 `main` push만으로 자동 반영된다.

## 작업 범위

- `.github/workflows/deploy.yml` 작성

---

## 세부 작업 목록

### 1단계: 워크플로우 파일 작성

- [x] `.github/workflows/` 디렉토리 생성
- [x] `deploy.yml` 작성
  - 트리거: `main` 브랜치 push
  - 실행 환경: `ubuntu-latest`
  - `environment: Production` 으로 Secrets 접근 범위 한정
  - `GCP_SA_KEY` Secret으로 gcloud 인증
  - `gcloud functions deploy --gen2` 배포 명령어 구성
    - 런타임: `nodejs20`
    - 리전: `asia-northeast3`
    - 엔트리포인트: `memoApi`
    - `--set-env-vars ALLOWED_IPS=${{ secrets.ALLOWED_IPS }}` 환경변수 주입

### 2단계: ALLOWED_IPS Secret 등록

- [x] GitHub `Production` Environment Secrets에 `ALLOWED_IPS` 추가 (쉼표 구분 IP 목록)
